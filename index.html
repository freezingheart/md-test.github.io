NFS创建用户在终端登录后自动挂载，退出时自动卸载。

title: NFS创建用户在终端登录后自动挂载，退出时自动卸载。
date: 2015-02-28 13:55:00

tags:

# useradd nfsuser

passwd nfsuser

然后进入其加目录/home/nfsuser，使用ls -al命令查看隐藏文件。
可以看到，有两个接下来需要用到的文件，.bashprofile和.bashlogout。
[info].bash_profile 类似于编程中的构造函数，当登录shell时，shell会寻找该文件做环境初始化。
.bashrc 是.bash_profile的替补。
.bash_logout 类似于编程中的析构函数，当登录shell退出时，shell会寻找该文件，并按其指示办事。[/info]
根据上面的解释，打开.bash_profile，直接在下方写入挂载的命令

sudo mount 192.168.2.1:/var/test /home/nfsuser/t

sudo是为了使用root权限挂载，普通用户是挂载不了的。
保存退出后，进入下一个.bash_logout这个文本，输入卸载的命令

sudo umount /home/nfsuser/t

保存退出后，输入visudo命令，来使mount开放给指定用户，
打开后找到root ALL=(ALL) ALL这一行，然后在下方添加两行，并写入内容。

nfsuser localhost=/bin/mount

nfsuser localhost=/bin/umount

然后即可exit退出当前登录用户来尝试nfsuser用户登录了。

小白向的系统重装教程

title: 小白向的系统重装教程
date: 2014-05-09 18:33:00

tags:

说明
==

很多新手在重装系统的时候都要找所谓的“高手”来帮忙重装系统，在大家眼中重装系统好像是件很高科技的事，其实重装系统很简单，从 Windows 的最初版本开始微软就在不停的改进系统安装的方法，每一代的系统比起上一代来说都更容易安装 / 使用。所以不用担心重装系统会很麻烦，你要做的就是把你电脑里你认为重要的东西、不想删掉的东西，复制到其他地方避免丢失，然后就可以跟着教程重装系统了。
在这个教程中，我会用尽可能容易理解的方式来教大家怎样重新安装操作系统。
我会尽可能把应该考虑的细节写下来，虽然可能有点啰嗦，不过很多东西只要大概了解下就好。

为什么要自己重装系统？
===========

电脑用了一段时间之后，常常会因为各种问题而恼火：功能不正常、操作卡顿、蓝屏死机、中了木马病毒……这些问题大多都能通过重装系统得到解决。
学会怎样重装系统之后，当你电脑崩溃的时候就不再需要到处求人了。而且现在电脑 90% 的问题都能通过重装系统解决，如果重装解决不了，那便很可能需要更换电脑中的零部件了。
自己重装的系统可以放心，这样的系统完整又干净，按教程说的装好系统也需要担心病毒木马。
另外还可以在女朋友 / 男朋友面前炫耀一把

决定好你要安装什么版本的系统
==============

先想好你要装什么系统，不然之后后悔是件麻烦的事。
建议的系统有：

> Windows 7 （推荐，发布时间较旧）
> Windows 8 （前两年发布的系统，采用了很多新技术，速度快，有的软件可能不能用）
> Windows 8.1 （是 Windows 8 的更新版本，更适合用鼠标操作）

决定之后你需要下载安装镜像，一个比较靠谱的下载地址是：http://msdn.itellyou.cn 例如需要安装 Windows 7，则在左边依次点击 操作系统 -> Windows 7 -> 简体中文 如果你有正版的序列号，可以选择序列号对应的系统； 如果你不是正版用户，那么就选择一个功能全一点的。

Windows 8.1系统中wfpdiag.etl 写入量非常大的解决方法

title: Windows 8.1系统中wfpdiag.etl 写入量非常大的解决方法
date: 2015-04-12 17:15:00

tags:

最近下载量比较大，虽然数据都是往机械硬盘写的，但是主硬盘（SSD）写入量还是很大，我的浦科特可禁不起这种折腾，一开始还以为是IDM的锅，然后把IDM的缓存文件路径改为D盘（机械硬盘）以后，情况还是没有好转，打开了资源监视器，调节至磁盘活动 按总数（字节/秒）排序 发现System进程读写C:\Windows\System32\wfp\wfpdiag.etl 的一个文件读写量非常大，排名第一，速百度之，发现这个文件是Windows创建的用于跟踪文件，收集IPsec和防火墙活动的诊断信息。也提供了关闭的方法
Win+X+A （以管理员方式运行命令提示符）同意UAC后，输入

netsh wfp set options netevents = off

返回成功
这时再看资源管理器，SSD的读写占用已经明显低下来了。

Ubuntu 14.10 Unity侧栏消失的解决方法

title: Ubuntu 14.10 Unity侧栏消失的解决方法
date: 2015-06-07 00:04:00

tags:

之前做了一个vnc2rdp，然后Unity就迷一样的返回了经典模式，侧边栏消失，要点左边的“活动”才出现，网上大多数解决方法是使用compizconfig-settings-manager(ccsm)来启用Ubuntu Unity Plugin 然而这并没有什么卵用，还是以前的界面，注销重登也不管用，现用重置compiz的方法成功了。

dconf reset -f /org/compiz/

重启Unity

setsid unity

然后就看到曾经熟悉的桌面了

将Typecho安装在二级目录下导致伪静态开启不成功

title: 将Typecho安装在二级目录下导致伪静态开启不成功
date: 2015-05-31 21:54:00

tags:

之前在实验环境搭了一个Typecho，用的lnmp.org的一键安装环境，然后就直接解压到www目录内，导致了访问出现了二级目录，也就是http://ip/default这样的URL，于是呢，今天想测试一下Nginx环境下的伪静态，然而并不行，测试许久之后，终于在[某个论坛][1]发现了自己再二级目录下，遂改正，之后便毫无问题。
顺便献上那位的代码：

location /子目录名-改成自己的/ {
if (-f $request_filename/index.html){
rewrite (.*) $1/index.html break;
}
if (-f $request_filename/index.php){
rewrite (.*) $1/index.php last;
}
if (!-f $request_filename){
rewrite (.*) /子目录名-改成自己的/index.php last;
}
}
