在CentOS Linux 5.5中完成 Samba 共享服务的配置

title: 在CentOS Linux 5.5中完成 Samba 共享服务的配置
date: 2015-02-09 11:37:00

tags:

本机环境：
宿主机OS：Windows 8.1 Pro x64
虚拟机软件：Oracle VituralBox 4.3.20
虚拟机OS：CentOS Linux 5.5

原题如下：
（二）在主机Centos-A1中完成Samba共享服务器的部署 (25分)
1、在此服务器中安装配置Samba服务，创建三个用户m1,m2,m3。分别建立共享m1,m2,m3,public，本地目录分别为/opt/a1, /opt/a2, /opt/a3, /opt/public；(10分)
2、默认以匿名访问，可以对public有读权限。进入其它文件夹时需要对其身份认证；(2.5分)
3、其中，m1用户属于manager组，对m1,m2,m3共享有读写权限。m2,m3为同一项目组m2的成员，可以互相对彼此文件有读的权限。/opt/a1的共享只有manager组用户可以访问；(10分)
4、提取本机eth0网卡IPv4地址，如（192.168.10.10），请把命令存入本机桌面BACKUP文件夹中command文件；（2.5分）


详解如下：

  1. 建立虚拟机并安装Guest OS（此处略）。不过，一个比较重要的问题是，在安装时要选中启用I/O APIC 选项（如下图），

    不然会在安装时，出现NET: Registered protocol family 2 显示并卡住而使安装无法正常进行（如下图）。

  2. 题1：在此服务器中安装配置Samba服务，创建三个用户m1,m2,m3。分别建立共享m1,m2,m3,public，本地目录分别为/opt/a1, /opt/a2, /opt/a3, /opt/public；(10分)
    a)

    yum install samba

使用在线安装或本地yum源安装（本地安装可以参考我之前的一篇文章《配置本地yum更新源》）。
↓由于我安装时使用Server模式安装，所以已经预装了Samba。

b)    mkdir /opt/public 
c)    mkdir /opt/a1
d)    mkdir /opt/a2
e)    mkdir /opt/a3

分别将匿名用户访问目录，m1,m2,m3用户访问目录建立。

f)    useradd m1 && passwd m1
g)    useradd m2 && passwd m2
h)    useradd m3 && passwd m3

将m1,m2,m3用户建立并设立密码

i)    smbpasswd –a m1
j)    smbpasswd –a m2
k)    smbpasswd –a m3

为m1,m2,m3用户建立smb用户，使其可以访问samba资源。


l) 

cd /etc/samba/

切换目录至/etc/samba/

m) 

vim smb.conf

其中：
i. 在[global]节下，添加map to guest = bad user 行。

ii. 在配置文件最后，添加[public]节
在节内添加以下行

path = /opt/public     //共享路径
browseable = yes      //是否可浏览
guest ok = yes        //是否可匿名访问


iii. 现在已经完成了匿名用户的配置。可以在Windows下使用\ip\public访问Samba共享资源。

iv. 在配置文件中，新建[a1]节，在节中增加以下行：
path = /opt/a1
browseable =yes
guest ok = no
writable =yes
valid users =m1 ,@manager //允许访问的用户


v. 依据[a1]节，新增[a2]、[a3]节
[a2]

path = /opt/a2
browseable =yes
guest ok = no
writable =yes
valid users =m2 ,@manager,@m2
read list = m3                   //表示在可读写环境中只读的用户

[a3]

path = /opt/a3
browseable =yes
guest ok = no
writable =yes
valid users =m3 ,@manager,@m2
read list = m2


n) 切换目录至/opt/
cd /opt/
o) 切换拥有者、拥有组
chown m1.manager a1 && chown m2.manager a2 && chown m3.manager a3
p) 为文件夹授权
chmod 775 m1 && chmod 775 m2 && chmod 775 m3

  1. 默认以匿名访问，可以对public有读权限。进入其它文件夹时需要对其身份认证；(2.5分)
    设置选项为smb.conf中的map to guest =bad user ,//设置匿名访问的一个重要点
    Security=user,
    及各个共享节中的valid users 。
  2. 其中，m1用户属于manager组，对m1,m2,m3共享有读写权限。m2,m3为同一项目组m2的成员，可以互相对彼此文件有读的权限。/opt/a1的共享只有manager组用户可以访问；(10分)
  3. 设置项为smb.conf中的各个共享节中的valid users
    例如[a2]共享节 ，valid users =m2 ,@manager,@m2
    表示可访问用户为m2,manager组用户,m2组用户。
  4. 提取本机eth0网卡IPv4地址，如（192.168.10.10），请把命令存入本机桌面BACKUP文件夹中command文件；（2.5分）
    ifconfig eth0
    在inet addr段中获取ip，保存至指定位置。
